CS21120 worksheet 1: Priority Queues

Priority Queue

 

We have seen that a queue structure ensures that customers are served in the order that they arrive. There are however occasions where we need to assign priorities so that the processing is not simply first come, first serve. Consider a hospital waiting room for example. In this case the staff will handle an emergency even if it is not first in line.

 

A priority queue is a data structure that stores data in a similar fashion to a queue except that higher priorities are placed at the front of the queue. Items that have the same priority still follow the "first in, first out" rule.

 

Consider how you might implement such a queue.

If any more broken arms arrive (assuming equal severity) they will be added to the back of the queue. In this case our Priority Queue ADT will work in the same way as the Queue ADT. But where will a broken neck be placed? We need to change our insert method to ensure that higher priorities are inserted before lower priorities.

 

 

  1. In order to store data in your priority queue, there must be some way of determining its priority. You need to define a suitable interface to support this.
  2. You now need to look again at our original Queue class. Supporting a Priority Queue using arrays is too restrictive. Not only would it have the disadvantage of being a fixed size but also you would need to do a lot of shunting as items are inserted into the middle of the structure. What you need is a more dynamic implementation. Vectors would be a useful method but are too language specific. You need to look at a linked list implementation.

 

Worksheet Tasks

1.                 Implement a priority queue

2.                 Test using the following scenario

 

Consider the design of a simulation program for an airport. Assume that it has only one runway (but try and design it with extensions in mind). There will be a queue of planes waiting to land and a queue of planes waiting to takeoff. Only one plane can use the runway at a time. Assume that all takeoffs take the same amount of time. Assume that all landings take the same amount of time but this might not be the same as take off. Assume that planes arrive for landing at random times but with a specified probability (e.g. 1 in 6mins, 1 in 10 mins etc.) Assume similar conditions for departure though the probability may be different.

Planes waiting to land take priority over planes waiting to take off.

 

The program should simulate planes arriving and departing.  Planes added to the arrivals queue should have a random time generated which specifies how long that they can remain in the air before they run out of fuel.

 

If you can implement the above, move on to develop an AWT to give a graphical display of the simulation.
