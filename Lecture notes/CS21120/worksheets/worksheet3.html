CS21120 Worksheet 3: Rational Rose, First Contact

 

Introduction

The aim of this exercise is to enable you to acquire the ability to start Rational Rose and use its more basic editing functions.

 

Steps

  1. Start an xterm on stonkin. Set the DISPLAY to the machine you are using. From the xterm, start Rational Rose with the command rose. Ignore the initial set of warning messages that you get the first time that you run Rose. These should not appear on subsequent runs. You should not need to change your path environment variable to access Rose.
  2. When the Rational Rose main window has appeared, select "Rational Rose Help Topics" from the "Help" menu. Open the "Rational Rose" topic, the open "Concepts", and then display "Analysis and Design".
  3. Work through the help pages and, at the same time, experiment with the diagramming facilities of the tool. In particular, try to construct a very simple use case diagram.

 

Note

There appears to be a bug in the SPARC version of Rational Rose that has prevented some students saving their work to file. We are investigating this with Rational. For the moment, save your work to files in /tmp. Use filenames containing your userid to avoid name conflicts with other students. At the end of your Rational Rose session, move the files from /tmp into your own filestore. (N.B. If you don’t do this, your files may be lost.) Before starting another Rational Rose session, move your files back from your own filestore to /tmp.
