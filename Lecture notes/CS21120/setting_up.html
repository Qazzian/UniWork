Setting up your machine

 

(In all cases aber/util directory structure should be copied from /dcs/mbr/JavaNotes)

 

In UNIX, you need to add the following to your .bashrc file:

CLASSPATH=".:/usr/local/lib/java_library:/usr/local/lang/java/lib"

export CLASSPATH

 

The above assumes that /usr/local/lib/java_library contains aber/util

 

In Windows’95, you need to do the following:

  • Create a file, C:\jdk1.1.6\rc.bat containing the following:

@echo off

set PATH=C:\jdk1.1.6\bin;%PATH%

set JAVA_COMPILER=symcjit

set CLASSPATH=.;C:\jdk1.1.6;C:\jdk1.1.6\lib\classes.zip;

doskey

The above assumes that you have copied the directory aber/util into C:\jdk1.1.6

 
  • Now create a shortcut to the DOS prompt icon on the desktop

Click the right button over the icon, open Properties and select Program

Give it a name, for example, "Java Development"

Set Working to C:\CSM1020 (or something similar for you to keep your java files)

Set Batch file to C:\jdk1.1.6\rc.bat

Set Run to Normal Window

Select an appropriate icon

 

In Windows NT

  • Start the Control Panel, select System, then edit the environment variables exactly as shown:

Variable: Path

Value: C:\jdk1.1.6\bin;%Path%

Variable: CLASSPATH

Value: .;C:\jdk1.1.6;C:\jdk1.1.6\lib\classes.zip;
  • Create a shortcut to the DOS prompt icon on the desktop

Edit Properties by clicking right mouse button while cursor is located over icon

Under Shortcut, edit Start in to specify the location of your working directory (i.e. where you store your java files). Select Apply.

Under Layout, change height to, say, 80 in order to give you a scroll bar. Select Apply.

The above assumes that you have the directory aber/util in C:\jdk1.1.6

 

And now back to writing and compiling the Java programs

  • Edit a file using gvim making the name of the file the same as the class but with the .java extension.

Example

If you are entering the class Radio then call this file Radio.java
  • Compile this code using the java compiler e.g.: javac Radio.java

(Note the use of the upper case R - even if windows ignores it, still use it!)

This generates bytecodes that are machine instructions for the java virtual machine. This can run on a variety of different platforms.
  • Now enter the RadioDemo class into another file RadioDemo.java
  • Compile this code using the java compiler: javac RadioDemo.java
  • Run the program using the java interpreter: java RadioDemo

The interpreter inspects and deciphers the bytecodes to ensure that it hasn’t been tampered with. It then loads any required classes and executes the actions that they specify.
