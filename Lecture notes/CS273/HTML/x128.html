2. Creating a simple DB

The "Whole Picture Exercise" (see "Overview") provides suitable examples for this. You might like to follow the general drift of that exercise but don't feel bound to do it all or to follow it in all details.

A suggested list of things to do is:

  1. Create the operator and tour tables with the fields specified. If you feel that you can apply the domain constraints, have a go, but don't get too tied up in it. You will not be able to do the Lookup Wizard bits.

  2. Create the primary and foreign keys.

  3. Import the data. You might do this manually. Alternatively, you could retrieve the tab separated data from the rtf of the worksheet, store it in a tex file (e.g. tour-data.txt) and then import it at the psql prompt by

    \copy tour from tour-data.txt

    See on-line manual for more details.

  4. Skip to the Creating Queries section and try to implement the following queries using select statements:

    1. Restrict operator to CompanyName and Telephone;

    2. Sort operatorin reverse alphabetical order by CompanyName;

    3. Create a query, adding both tables. Show the CompanyName, Start, Visiting and StandardFare fields. Show the fares in Ascending order and subsort by CompanyName
