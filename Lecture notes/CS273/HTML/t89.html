1. Postgres

Postgres is an object-relational database management system. We will, initially, use only the subset of its facilities which make it appear to be a relational DBMS. It is public domain and open source.

The Department of Computer Science manages an installation of Postgres on one of its Sun machines. The hostname of this machine is database.dcs.aber.ac.uk and it can be accessed via a variety of mechanisms including ODBC, JDBC, a command line interpreter (monitor) called psql and a simple GUI called pgaccess.

Full documentation for Postgres and its local installation can be found on the Department's web. These pages are maintained by the support group. You should seek help and examples here for using psql or pgaccess for testing; the JDBC interface and individual SQL statements. Note that many SQL statements have minor extensions to or variations from any SQL standard.

For an understanding of later sections of this document, it is necessary to know that psql can receive both SQL statement and psql commands. These latter are preceded by a backslash. They provide database listing operations and interaction with the operating system. See the on-line manual for details.
