WaitPlayer.java

WaitPlayer uses a slightly different approach to deal with the asynchronous events used by the JMF. Try compiling and running WaitPlayer. You will also notice that, although it does not ``autoplay", it does always display the first frame of the movie. Look at the Java code and see if you can spot why.

You will also notice that I've added a ``Rewind and Play" button at the top of the display. What happens if you click it while the movie is playing? Look at the code and discover why it behaves like that, and how it achieves the rewind and play action when allowed.
