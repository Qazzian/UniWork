Media Conversion

As you saw above, the movie kayaker.avi would not show video on your Intel/Solaris workstation. Use Browser to remind yourself what happened. Use the Plugin Viewer to see what components have been used to make the player and thus what media streams must be included in the movie.

Now, look at the code of Convert.java. You will notice that it is clearly trying to make efforts to convert the kayaker.avi file into a different format. Compile and run the code on your Intel/Solaris workstation.

What happens?

It probably apparently executed quite quickly. Try to play the newfile.mov file that it created. Does it work? No? What went wrong then ? Look at the output that was generated and the jmf.log file produced. Have you spotted the problems?

Now, start a window on stonkin and run Convert.java again in that. What happens?? Click with the left button on the right hand symbol that looks a bit like frames of a film. An information box will pop up. Investigate what that is telling you. Spotted it ?? Yes, its running, but taking an absolute age. If you have got the patience it will eventually convert the file into a new format. It will probably take five minutes or more!

I cooked one earlier :-) and if you access the URL

http://www.aber.ac.uk/dap/JMF/newfile.mov

using (say) Browser, then you should be able to both watch and hear the canoe going over the waterfall.
