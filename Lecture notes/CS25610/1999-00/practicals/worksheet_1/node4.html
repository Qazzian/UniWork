Client connection and disconnection

The protocol is very simple. The server is sitting listening for incoming TCP calls on port number 4000. On receiving an incoming call, the server will extract one line of text from the socket, split the line of text into three tokens and use these to decide how to respond. The server will then send some messages back to the client and then close the socket. If the client wishes to send another command line into the server, then the client must open another TCP connection to the server and send the new command on that new connection.
